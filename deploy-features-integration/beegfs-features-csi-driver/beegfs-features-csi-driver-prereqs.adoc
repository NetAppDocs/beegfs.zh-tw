= 部署 BeeGFS CSI 驅動程式的前提條件
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
在部署和設定 BeeGFS CSI 驅動程式之前，請確保您有一個正在執行的 Kubernetes 叢集、可以存取您的 BeeGFS 集群，以及安裝叢集範圍資源的適當權限。



== 驗證 Kubernetes 設定

執行以下命令檢查 Kubernetes 叢集中 Pod 的狀態：

[source, console]
----
kubectl get pods
----
**預期輸出：**

[listing]
----
root@node01:/etc/cni/net.d# kubectl get pods
No resources found in default namespace.
----
如果沒有 pod 正在運行，則清單為空。否則，pod 應該能夠正常列出。



== 驗證 BeeGFS 用戶端和服務

執行以下命令驗證 BeeGFS 叢集是否運作正常且所有服務均可存取：

BeeGFS 7.4

[source, console]
----
beegfs-check-servers
----
**BeeGFS 8**

[source, console]
----
beegfs health check
----

NOTE: CSI驅動程式可與兩者搭配使用 `beegfs-dkms-client`和 `beegfs-client`必須將其安裝在每個 Kubernetes 節點上。有關安裝步驟，請參閱 BeeGFS 用戶端 DKMS 文件： https://doc.beegfs.io/latest/advanced_topics/client_dkms.html[]
